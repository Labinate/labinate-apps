What to name your resume file

It's the last thing you do before applying, and most people do it on autopilot — hit "Save As," accept whatever's in the box, and send. But your file name is the very first thing a recruiter sees, before a single word of the resume itself. "Jordan-Lee-Resume.pdf" and "resume-final-v3.docx" tell two different stories, and it takes ten seconds to send the right one.

Updated July 2026 · ~5 min read · by Labinate

Here's the whole guide in one line: name it FirstName-LastName-Resume.pdf. Everything below is why that format wins, the traps that make people send something worse, and the handful of times it's worth adding more.

The formula

A good resume file name does one job: it identifies you at a glance when your file is sitting in a folder next to two hundred others. That means it needs your name and the word "resume," and nothing that only makes sense to you.

The default that always works Jordan-Lee-Resume.pdf

Full first and last name, the word "Resume," a PDF extension. That's it. When a recruiter downloads a batch and sorts the folder by name, yours is instantly findable and unmistakably yours. When they forward it to a hiring manager, the file arriving in an inbox already says who it's from. No guessing, no "resume(3).pdf," no opening it just to find out whose it is.

What not to name it

The bad names aren't random — they're the defaults and the working labels that leak out when you're moving fast. Each one costs you a little.

✕ Don't send

  • resume.pdf
  • my-resume.docx
  • resume-final.pdf
  • resume-final-v3.docx
  • resume-draft.pdf
  • CV_2026_new(2).pdf
  • Untitled.pdf

✓ Send instead

  • Jordan-Lee-Resume.pdf
  • Jordan-Lee-Resume.pdf
  • Jordan-Lee-Resume.pdf
  • Jordan-Lee-Resume.pdf
  • Jordan-Lee-Resume.pdf
  • Jordan-Lee-Resume.pdf
  • Jordan-Lee-Resume.pdf

Two of these deserve a special warning. "resume.pdf" is the most common file name in every recruiter's downloads folder, so yours immediately collides with a dozen identical ones — some systems even overwrite the earlier download silently. And "resume-final-v3" (or "-draft," or "-NEW") quietly tells the reader you didn't do the last ten-second check before sending. Version labels belong on the copies on your own drive, never on the one that leaves it.

Hyphens, not spaces

Use hyphens to separate the words. Spaces mostly work now, but they turn into %20 in email links and download URLs, look messy in a file path, and still trip up the occasional older upload form. Underscores are fine too — Jordan_Lee_Resume.pdf — they just read a touch heavier than hyphens. The one rule that matters: pick one separator and use it everywhere, including your cover letter.

✓ Clean separators

  • Jordan-Lee-Resume.pdf
  • Jordan_Lee_Resume.pdf

Riskier

  • Jordan Lee Resume.pdf (→ %20)
  • JordanLeeResume.pdf (hard to read)

When to add the job title

If you're tailoring your resume to a specific role — and for the jobs you actually want, you should be — it's worth adding the target title to the file name. It keeps your own versions straight and gently signals the role you're aiming at.

Tailored version — role added Jordan-Lee-Resume-Product-Manager.pdf

Keep it to the role, not a sentence. Adding the company name is optional and carries a real risk: the classic mistake is sending "Jordan-Lee-Resume-Acme.pdf" to a job at Northwind because you forgot to rename it. If you include the company, treat the file name as one more thing to check before you attach it. When in doubt, the plain FirstName-LastName-Resume.pdf is never wrong.

The tailoring trap

You keep one master file and "Save As" a new name per application. Great for staying organized — dangerous at send time, because it's easy to attach yesterday's company-stamped version. Open the attachment from the email itself and read the file name once before you click send.

PDF or Word — and the extension

The base name doesn't change with format; only the extension does. Send a PDF by default (.pdf) so your layout can't shift on someone else's machine, and switch to .docx only when a posting or recruiter explicitly asks for an editable file. Either way the name stays the same: Jordan-Lee-Resume.pdf or Jordan-Lee-Resume.docx. For the full trade-off, see PDF or Word resume.

Match your cover letter

If you're sending a cover letter too, name it the same way so the two files sit together and read as a set:

A matched pair Jordan-Lee-Resume.pdf
Jordan-Lee-Cover-Letter.pdf

Same name, same separator, same extension — swap only the document word. When both land in a recruiter's inbox, they're obviously from the same organized candidate. Then attach them the right way when you email your resume.

Quick reference

  • Default → FirstName-LastName-Resume.pdf.
  • Tailored → add the role: FirstName-LastName-Resume-Job-Title.pdf.
  • Separator → hyphens (or underscores), never spaces.
  • Extension → .pdf by default, .docx only on request.
  • Cover letter → same name, swap "Resume" for "Cover-Letter."
  • Never → resume.pdf, resume-final-v3, draft, NEW, or a leftover company name.
The whole thing in one breath. Name it FirstName-LastName-Resume.pdf — your real name, the word "Resume," hyphens between words, a PDF extension. Add the job title only when you've tailored the version, keep version numbers and the wrong company off the copy you send, match the cover letter's name to it, and read the file name once from inside the email before you attach. It takes ten seconds and it's the first thing anyone sees.

Frequently asked questions

What should I name my resume file?

Use your full name plus the word Resume: FirstName-LastName-Resume.pdf, for example Jordan-Lee-Resume.pdf. This is the format recruiters and applicant tracking systems expect — it identifies the file instantly when it's sitting in a folder of 200 others. Avoid generic names like resume.pdf or my-resume.docx, and never leave working labels like resume-final-v3 or resume-draft on the copy you send.

Should a resume file name have spaces, hyphens, or underscores?

Use hyphens. Spaces can turn into %20 in email links and download paths and occasionally break older systems, so Jordan-Lee-Resume.pdf is safer than "Jordan Lee Resume.pdf". Underscores work fine too (Jordan_Lee_Resume.pdf) but hyphens read more cleanly. Pick one separator and use it consistently across your resume and cover letter.

Should I put the job title or company in my resume file name?

Only if you're tailoring a version to a specific role, and even then keep it short: Jordan-Lee-Resume-Product-Manager.pdf. Adding the target title helps you keep your own files straight and signals the role you're after. Adding the company name is optional and easy to get wrong — the real risk is forgetting to change it, so if you include it, double-check it matches the posting before you hit send.

Does the resume file name actually matter to an ATS?

It matters more to the human than the machine, but both benefit. Most applicant tracking systems read the file's contents, not its name, so a good name won't boost your keyword match. What it does is make you look organized the moment a recruiter downloads a batch of resumes — yours says "Jordan Lee" while others say "resume(3).pdf." That small signal of care is free, so there's no reason not to claim it.
