How to send a self-destructing message

Sometimes you want to send something — a password, a code, a private note — that shouldn't outlive the moment it's read. A self-destructing message does exactly that: it shows once, then it's gone. Here's how they work and how to send one for free, with no app and no account.

Updated July 2026 · ~4 min read · by Labinate

Every message you send in email or chat is, by default, permanent. It sits in your Sent folder, the recipient's inbox, both sides' backups, and often a company server — long after the thirty seconds you actually needed it. A self-destructing message flips that default: the content is designed to vanish after a single read, so there's nothing left to leak later.

The best versions do three things at once: encrypt the message so no server can read it, make it openable only once, and store nothing permanently.

How to send one in four steps

  1. Write your message. Open a zero-backend self-destructing message tool and type what you need to send — a password, a code, a private note.
  2. Generate a one-time link. The tool encrypts your message right in the browser and hands you a link. The decryption key rides inside the link itself, not on any server.
  3. Send the link. Share it however you like — chat, email, SMS. Even if that channel keeps a log, the log only holds a dead link once the message has been opened.
  4. It burns after reading. The moment your recipient opens it, the message is shown once and destroyed. Reload the link and there's nothing there.

When to use a self-destructing message

  • Passwords and logins you're handing to a colleague, client, or family member.
  • 2FA backup codes or recovery keys that must never sit in a chat history.
  • API keys, tokens, and secrets during a quick hand-off between teammates.
  • A private address, number, or one-off note you'd rather not leave a permanent trace of.
When not to. For credentials your team reuses every day, a shared password manager is a better fit — one-time links are for hand-offs, not for storage. And remember a recipient can always screenshot: self-destruction protects against lingering copies, not against a determined reader.
