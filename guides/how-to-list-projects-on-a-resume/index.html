How to list projects on a resume

A projects section is one of the most underused parts of a resume — and one of the most powerful when your job history is thin or you want to prove a skill a title can't. The catch is that a project only helps if it reads like real, describable work. Here's when to include one, where it goes, and exactly how to write the bullets, with a full example to copy.

Updated July 2026 · ~5 min read · by Labinate

Most people either leave projects off entirely or dump them in as vague one-liners — "personal website," "class project," "app I built." Neither helps. Done right, a projects section turns a class assignment, a weekend build, or a piece of freelance work into evidence that you can do the job. That's especially valuable if you're a student, a career changer, or in a field where what you make matters more than where you've worked.

When to include a projects section

A projects section earns its space only when it shows something your work experience doesn't. Include one when:

  • Your work history is thin. Students, new grads, and people writing a resume with no experience can use projects to prove skills before they have a job title to point to.
  • You're changing fields. A side project in the target field is the fastest way to show you can already do the new work, not just want to.
  • You're in a build-something field. In software, design, data, and marketing, a portfolio piece or shipped project is often stronger evidence than a bullet about a past role.
  • A specific project maps to the posting. If you built something that uses the exact skill the job asks for, it belongs on the page.

If your experience already demonstrates the same skills, skip the section and put that space toward stronger work bullets. A projects section is worth it only when it adds new evidence.

Where the section goes

Placement follows one rule: your strongest material goes where the recruiter sees it first. So it depends on how much your projects are carrying.

  • Projects are your main evidence (students, career changers) — put the section high, right after education, above or in place of a short work-experience section.
  • Projects are a bonus (experienced professionals) — place them below work experience, as supporting proof of range or initiative.

Either way, give it a clear heading — Projects, Selected Projects, or Personal Projects — so a resume parser and a human both know what they're looking at.

How to write a project bullet

This is where projects live or die. Write each one exactly like a job bullet: a strong action verb, what you built, the tools or methods you used, and the result. The difference between a project that impresses and one that gets skipped is entirely in the phrasing.

Weak

Made a personal website to practice coding.

Strong

Built and deployed a portfolio site in React and Tailwind; automated builds with GitHub Actions, cutting deploy time to under a minute.

The strong version names concrete tools and attaches a measurable result. Wherever you can quantify anything — users, hours saved, size, percent, dollars — do it. Numbers turn a hobby into output.

What each project entry should contain

Keep every entry to a consistent shape so the section scans cleanly:

  1. Project name and type — e.g. "Campus Events Finder — Personal Project" or "Sales Dashboard — Freelance."
  2. One line of context if it isn't obvious — what it does and who it's for.
  3. Two or three achievement bullets — built with, using, resulting in.
  4. A link — a live URL or repository, if you have one, so the reader can verify it.
Projects section Expense Tracker — Personal Project
  • Built a React + SQLite web app that categorizes spending automatically; used by 40+ people in the first month
  • Designed a REST API and deployed on a free tier with zero hosting cost
  • Live: expense-tracker.example.com · Code: github.com/you/expense-tracker
Sales Dashboard — Freelance
  • Built an interactive Looker Studio dashboard for a local retailer, cutting their weekly reporting time from 3 hours to 20 minutes
Pick two or three, not ten. A projects section is a highlight reel, not an archive. Choose the projects that best match the job description and are impressive enough to defend in an interview. Two strong, quantified projects beat five thin ones — and they keep you to one page.

Frequently asked questions

Should I include a projects section on my resume?

Include one when it adds evidence a job title can't — if your work history is thin, you're changing fields, you're in a build-something field like software or design, or a specific project maps to the posting. If your experience already covers the same skills, spend the space there instead. A projects section is only worth it when it shows something new.

Where does the projects section go on a resume?

Wherever your strongest material should be seen first. If projects are your main evidence (students, career changers), put the section high — right after education, above a thin work-experience section. If they're a bonus for an experienced candidate, place them below work experience.

How do I write a project bullet on a resume?

Use the job-bullet formula: an action verb, what you built, the tools you used, and the result. "Built a React expense tracker with a SQLite backend, used by 40+ people" reads like output; "made an app" does not. Name concrete tools and attach a number wherever one honestly exists.

Can personal or side projects count as experience?

Yes. A personal app, a freelance build, a class project, a hackathon entry, or an open-source contribution is real, describable work, and recruiters treat it that way — especially early in a career. Write each with achievement-style bullets and include a live link or repository when you have one.
